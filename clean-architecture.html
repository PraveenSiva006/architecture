Entities

Core business objects and rules. Pure logic — no framework, no database, no UI. The most stable layer.

Application-specific business rules. Orchestrates the flow of data to/from entities. Independent of UI and DB.

Interface Adapters

Converts data between use cases and the external world. Controllers receive input; presenters format output.

Infrastructure

Frameworks, databases, UI, external services. Details that are easy to swap. Should know nothing of inner layers.

⬡ The Dependency Rule

Source code dependencies must point only inward. Nothing in an inner circle can know anything about something in an outer circle. The name of a framework, a database, or a UI component must not be mentioned by the code in the inner rings. Entities know nothing. Use cases know only entities. Adapters know use cases. Infrastructure knows everything — but it's the detail that changes.
